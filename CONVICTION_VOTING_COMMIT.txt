feat: implement tip conviction voting

Implement a comprehensive conviction voting system where voting power accumulates
over time for funding decisions. This feature enables long-term commitment-based
governance with time-weighted voting power.

## Features Implemented

### 1. Conviction Accumulation
- Time-based voting power multiplier (1x to 3x over 30 days)
- Linear interpolation for smooth conviction growth
- Accumulated conviction tracking across all proposals
- Configurable conviction period and maximum multiplier

### 2. Voting Power Calculation
- Effective voting power = base_voting_power × conviction_multiplier
- Conviction multiplier grows from 1x to max over conviction_period
- Real-time calculation based on time locked
- Support for fractional multipliers using fixed-point arithmetic

### 3. Proposal Thresholds
- Reduced proposal creation threshold based on voter conviction
- Maximum 50% threshold reduction for high-conviction voters
- Formula: adjusted_threshold = base_threshold × (1 - conviction_bonus)
- Encourages long-term governance participation

### 4. Vote Changes
- Support for changing votes with decay penalty
- Accumulated conviction decays when vote changes
- Decay rate: 0.01% per second (configurable)
- Discourages frequent vote changes

### 5. Conviction History
- Persistent tracking of all conviction votes
- Historical records for auditing and analysis
- Supports conviction voting analytics
- Enables governance participation metrics

## Technical Implementation

### New Modules
- `governance/conviction.rs`: Core conviction voting logic
- `governance/conviction_integration.rs`: Integration with existing voting system

### Core Functions
- `init_conviction_voting()`: Initialize system with default config
- `cast_conviction_vote()`: Cast a conviction vote on proposal
- `change_conviction_vote()`: Update existing conviction vote with decay
- `calculate_conviction_multiplier()`: Calculate time-based multiplier
- `calculate_effective_voting_power()`: Get voting power with multiplier
- `get_voter_total_conviction()`: Track total conviction across proposals
- `get_adjusted_proposal_threshold()`: Calculate reduced threshold

### Data Structures
- `ConvictionVote`: Stores voter's conviction vote details
- `ConvictionConfig`: Global configuration parameters
- `ConvictionVotingDetails`: Query result with all voting metrics

### Storage Keys
- `ConvictionVote(proposal_id, voter)`: Current conviction vote
- `ConvictionConfig`: Global configuration
- `ConvictionHistory(proposal_id, voter)`: Historical records
- `VoterConvictionTotal(voter)`: Total conviction per voter

### Events
- `("conv_vote",)`: Emitted when conviction vote is cast
- `("conv_chg",)`: Emitted when conviction vote is changed

## Configuration

Default parameters:
- Conviction Period: 30 days (2,592,000 seconds)
- Max Conviction Multiplier: 3x (3,000,000)
- Conviction Decay Rate: 0.01% per second (100 bps)
- Min Conviction Threshold: 0.1 tokens (100,000)

All parameters are configurable by contract admins.
